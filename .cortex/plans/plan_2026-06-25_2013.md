## Context

NY Taxi data warehouse. 10 Bronze source nodes. Existing V1 node types: Stage (SILVER), Dimension/Fact/View/SemanticView (GOLD). All new nodes will be V1 YAML format using `ref()` for joins.

## Implementation Steps

### Silver Layer — Staging Nodes (7 nodes)

1. **SILVER-STG_TRIPS.yml** — Stage, UNION ALL of CAB_TRIPS + YELLOW_CAB_TRIPS with data type casting (FLOAT→NUMBER for IDs, NULL handling via NVL)
2. **SILVER-STG_VENDOR.yml** — Stage, joins VENDOR + VENDOR_DETAILS on VENDOR_ID
3. **SILVER-STG_LOCATION.yml** — Stage from LOCATION (passthrough + NVL)
4. **SILVER-STG_PAYMENT_TYPE.yml** — Stage from PAYMENT_TYPE
5. **SILVER-STG_RATE_CODE.yml** — Stage from RATE_CODE
6. **SILVER-STG_DRIVER.yml** — Stage from DRIVERS
7. **SILVER-STG_CALENDAR.yml** — Stage from CALENDAR

### Gold Layer — Star Schema (6 nodes)

8. **GOLD-DIM_VENDOR.yml** — Dimension (SCD2), BK=VENDOR_ID, change-track VENDOR_NAME, HQ_ADDRESS, PHONE, DRIVER_COUNT
9. **GOLD-DIM_LOCATION.yml** — Dimension (SCD2), BK=LOCATION_ID, change-track BOROUGH, ZONE, SERVICE_ZONE
10. **GOLD-DIM_PAYMENT_TYPE.yml** — Dimension (SCD2), BK=PAYMENT_TYPE_ID, change-track PAYMENT_TYPE
11. **GOLD-DIM_RATE_CODE.yml** — Dimension (SCD2), BK=RATE_CODE_ID, change-track RATE_CODE
12. **GOLD-DIM_DRIVER.yml** — Dimension (SCD2), BK=DRIVER_ID, change-track name/license/phone/email/rating/is_active
13. **GOLD-FACT_TRIPS.yml** — Fact with business key TRIP_ID, FK lookups to all dimensions via LEFT JOIN on staging + dimension refs. Measures: TRIP_DISTANCE, FARE_AMOUNT, TIP_AMOUNT, TOLLS_AMOUNT, TOTAL_AMOUNT, PASSENGER_COUNT, surcharges.

### Gold Layer — Aggregation + Semantic (2 nodes)

14. **GOLD-AGG_TRIPS_DAILY.yml** — View (daily agg), joins FACT_TRIPS to DIM_LOCATION (pickup), aggregates by DATE, BOROUGH, ZONE: trip_count, total_revenue, avg_distance, total_passengers
15. **GOLD-SV_TRIPS.yml** — SemanticView (nodeType 364), sources from FACT_TRIPS + all dimensions. Defines facts, dimensions, metrics (avg fare, total revenue, trip count).
